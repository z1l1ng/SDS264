NOTES:

Most punctuation characters, like ., +, *, [, ], and ?, have special meanings and are called metacharacters.

. will match any character “a.” will match any string that contains an “a” followed by another character:

str_view(fruit, “a…e”) “a”, followed by three letters, followed by an “e”

? makes a pattern optional (i.e. it matches 0 or 1 times)

str_view(c(“a”, “ab”, “abb”), “ab?”) ab? matches an “a”, optionally followed by a “b” > [1] │ > [2] │ > [3] │ b

+ lets a pattern repeat (i.e. it matches at least once)

str_view(c(“a”, “ab”, “abb”), “ab+”) ab+ matches an “a”, followed by at least one “b” > [2] │ > [3] │

* lets a pattern be optional or repeat (i.e. it matches any number of times, including 0).

ab* matches an “a”, followed by any number of “b”s. str_view(c(“a”, “ab”, “abb”), “ab*“) > [1] │ > [2] │ > [3] │

Character classes defined by []

[abcd] matches “a”, “b”, “c”, or “d”.

[^abcd] matches anything except “a”, “b”, “c”, or “d”.

str_view(words, “[aeiou]x[aeiou]”) words containing an “x” surrounded by vowels > [284] │ ct > [285] │ mple > [288] │ rcise > [289] │ st

str_view(words, “[^aeiou]y[^aeiou]”) a “y” surrounded by consonants: #> [836] │ tem #> [901] │ e

alternation, |, to pick between one or more alternative patterns

str_view(fruit, “apple|melon|nut”) fruits containing “apple”, “melon”, or “nut” > [1] │ > [13] │ canary > [20] │ coco > [52] │ > [62] │ pine > [72] │ rock

str_view(fruit, “aa|ee|ii|oo|uu”) repeated vowel > [9] │ bld orange > [33] │ gseberry > [47] │ lych > [66] │ purple mangostn

#quantifiers {n} matches exactly n times. {n,} matches at least n times. {n,m} matches between n and m times.

parentheses have another important effect: they create capturing groups that allow you to use sub-components of the match

\1 refers to the match contained in the first parenthesis, \2 in the second parenthesis

str_view(fruit, “(..)\1”) any two but repeated > [4] │ ba > [20] │ nut > [22] │ mber > [41] │ be > [56] │ ya > [73] │ s berry

want to use parentheses without creating matching groups. You can create a non-capturing group with (?:)

x <- c(“a gray cat”, “a grey dog”) str_match(x, “gr(e|a)y”) > [,1] [,2] > [1,] “gray” “a” > [2,] “grey” “e”

str_match(x, “gr(?:e|a)y”) > [,1]
> [1,] “gray” > [2,] “grey”
